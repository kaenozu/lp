## 2. 背景

個人開発ポータルと各アプリを継続的に運営するにあたり、ブランド名、対象ユーザー、収益化方針、ストア上の対象年齢・カテゴリ方針を早期に固定する必要がある。

これらが未決定のままだと、以下の作業で判断が分岐し、手戻りが発生する:

- ポータルサイトの名称、ドメイン、ロゴ、OGP、ストア表記
- プライバシーポリシーや利用規約の記述
- アプリの画面設計、機能範囲、オンボーディング
- Google Play Consoleの対象ユーザー設定
- 広告、課金、データ保護に関する実装
- Families Policyを含む追加要件への対応
- 今後追加するアプリを含めたブランド拡張

初期公開では機能や収益化を広げすぎず、識別しやすいブランドの確立、主要ユーザーへの価値提供、品質および信頼性の検証を優先する。

## 3. 決定事項一覧

### a. ブランド名に「アプノキ」を採用する

**決定内容**: 個人開発ポータルおよび開発活動全体の正式ブランド名として「アプノキ」を採用する。

- 正式表記: アプノキ
- ローマ字表記: apunoki
- 移行期間中のサイトタイトル: `アプノキ｜個人開発の小さなアプリ`
- タグライン候補: `小さなアプリを、ひとつずつ。`
- 「小さなアプリ工房」はブランド名ではなく活動説明として継続使用可

**理由**:
- 4文字の造語で既存の一般語や類似サービスと区別しやすい
- 検索結果やSNSアカウントを将来的に占有しやすい
- 特定のアプリ、利用者層、技術分野に依存しない
- アプリ数やジャンルが増えてもブランド名変更不要
- apunoki.dev / .app / .com に取得余地あり

**判断基準**: 検索識別性、読みやすさ、URL/SNS/ストアでの扱いやすさ、将来の拡張性、衝突可能性、個人開発としての温かさ

---

### b. 「あしたもつもの」の対象ユーザーを保護者に限定する

**決定内容**: 初回公開の主要ユーザーを園児・児童・生徒の保護者とする。子供本人が操作する機能は初回公開に含めない。

保護者が以下の操作を行う前提:
- お便り・画像・スクリーンショットの取り込み
- OCRや抽出結果の確認・修正
- 持ち物・提出物・集金・予定の管理
- 通知時刻や子供情報の設定
- 完了状態の更新

**理由**: 現在の主要機能は保護者による操作に適している。子供向け機能追加はUI/権限/データ設計/通知/アカウント共有の要件が大幅に増える。保護者向けに限定することでオンボーディング・ストア説明・プライバシーポリシーの表現を一貫させられる。

---

### c. 「あしたもつもの」v1.0には広告を導入しない

**決定内容**: v1.0にはバナー・インタースティシャル・リワード広告を含む広告機能を導入しない。

初回公開で優先するもの:
- OCRおよび項目抽出の精度検証
- 共有メニューを含む取込経路の安定性検証
- データ消失、二重取込、誤抽出などの不具合確認
- プライバシーおよびデータ保護方針の検証
- 継続利用率と実利用上の課題の把握
- クラッシュ、ANR、端末依存問題の確認

**理由**: 学校/園のお便りには個人情報が含まれ得る。広告SDK導入で通信・同意管理・データセーフティ申告の検証項目が増える。初期利用者数が少ない段階では広告収益よりSDK保守コストの方が大きい。

---

### d. 「へぇの種」を一般成人向けとして提供する

**決定内容**: 主要対象ユーザーを一般成人とする。子供や家族での利用を禁止するものではないが、初回公開では子供を主要対象に設定せず、Families対応を前提とした設計・運用は行わない。

**理由**: 雑学クイズという形式だけを理由に子供向けとするとFamilies Policyの追加要件を背負う。成人向けにすることで問題テーマ・文章表現・難易度の自由度を確保できる。

---

## 4. 採用しない選択肢

| 選択肢 | 棄却理由 |
|--------|---------|
| 「小さなアプリ工房」継続 | 類似表現既存、識別性弱い、検索占有しにくい、ドメイン長い |
| 既存アプリ名に寄せる | 特定ジャンルに見える、将来の拡張性低い |
| プライマリドメインに .app | 活動全体の入口としては用途が狭い |
| プライマリドメインに .com | 個人開発・技術活動の性格が伝わりにくい |
| 親子共同利用アプリとして公開 | 追加設計・開発負荷大、初回公開の焦点から外れる |
| v1.0から広告導入 | 不具合と広告SDK問題の切り分け困難、UX阻害 |
| 「へぇの種」を子供向け公開 | コンテンツ管理負担大、SDK/収益化に制約 |

---

## 5. 今後の見直し条件

### ブランド名・ドメイン
- 商標調査で重大な競合が判明した
- SNS/ストア/検索で識別性を確保できない
- apunoki.dev の継続利用が困難になった
- 事業内容が根本的に変化した

### 「あしたもつもの」対象ユーザー
- 保護者から子供本人操作への要望が継続的に寄せられる
- 子供向け画面追加で明確な効果が期待できる
- 役割分離・操作制限・データ共有を安全に実装できる
- 保護者向け機能が十分に安定した

### 「あしたもつもの」広告・収益化
- 継続利用者数、OCR/AI実費、利用頻度、継続率、支払い意向が揃った段階で再評価

### 「へぇの種」対象ユーザー
- 子供/保護者からの利用実績と要望が十分確認された
- 子供向け問題セットを分離管理できる
- 使用SDK/外部リンク/収益化方式がFamilies要件に適合できる

---

## 6. 次のアクション

### P0: ブランド基盤の確定
- apunoki.dev を取得する
- apunoki.app / apunoki.com の防御的取得を判断
- 商標・検索・SNSで競合を最終確認
- GitHub Organization 命名方針を決める
- ブランド正式表記をリポジトリ内で統一

### P0: ポータルサイトの移行
- サイトタイトルを「アプノキ｜個人開発の小さなアプリ」に変更
- ブランド名、description、OGP、Twitter Cardを更新
- canonical URL, sitemap.xml, robots.txt を新ドメイン対応
- 旧URLからのリダイレクト方針決定
- Search Console / アクセス解析設定更新

### P0: 「あしたもつもの」v1.0方針反映
- ストア説明を保護者向けに統一
- 子供が主要利用者に見える表現を削除
- Play Console 対象ユーザー設定確認
- 広告SDK/コードがv1.0に含まれないことを確認
- プライバシーポリシーとデータセーフティ申告を照合
- 実機テスト完了
- 不要な同意画面・設定項目を整理

### P1: 「へぇの種」公開方針反映
- ストア説明を一般成人向けに統一
- 子供向け・知育・教育専用と誤認される表現を見直し
- Play ConsoleでFamilies対象外設定を確認
- 問題文・解説の事実確認完了

### P1: ブランド運用ルール
- ロゴ初版作成
- ブランド名・アプリ名・タグラインの使い分けを文書化
- README/LICENSE/プライバシーポリシー/サポートページの表記ルール統一
- 新規アプリ追加時の掲載テンプレート作成
