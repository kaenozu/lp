# 「小さなアプリ工房」全体実施計画

- 計画開始日: 2026年7月19日
- 計画期間: 2026年7月19日〜2026年12月20日
- 稼働前提: 個人開発、平日夜間＋週末、週8〜10時間
- 対象: ポータルサイト、「あしたもつもの」、「へぇの種」
- インフラ方針: 静的HTML/CSS/JavaScript、Flutter、ローカル保存、Cloudflare Pages
- 予算: 原則0円。カスタムドメイン費用のみ許容
- 最優先目標: 「あしたもつもの」のAndroid正式公開と、「へぇの種」を継続的に開発できる問題データ基盤の完成

---

## 0. 最初に決めること

以下の4項目を初期意思決定ゲートとする。
これらが決まるまでは、ドメイン購入、ストア素材作成、広告SDK導入、「へぇの種」の本実装を開始しない。

| 項目 | 決定 | 理由 |
|------|------|------|
| a. ブランド名 | 調査して正式決定 | 個人開発、小規模アプリ、複数作品をまとめるポータル |
| b. 「あしたもつもの」の対象 | 保護者のみ | 初期UX、プライバシー、ストア審査、通知設計を単純化 |
| c. 初回広告 | 入れない | 信頼性・精度・データ保護の検証を優先 |
| d. 「へぇの種」の対象 | 一般成人向け | Families対応を初期から背負わない |

### 完了条件（2026/07/26まで）

- docs/decisions/0001-initial-product-decisions.md に記録
- ブランド名の採否、採用ドメイン、各アプリの対象ユーザー、広告方針を文書化
- 判断理由と将来見直し条件も記載

## 5. 進行管理ルール

- **WIP制限**: Doingは主要タスク1件まで（30分以内の細かいタスクは並行可）
- **週次レビュー**: 毎週日曜30分。完了/未完了/遅延理由/新P0問題/翌週計画
- **タスク完了定義**: 実装完了＋正常系確認＋異常系確認＋テスト＋lint/静的解析/ビルド＋ドキュメント更新＋本番確認
- **遅延時の削る順序**: ポータル装飾 → 分析拡張 → 問題数縮小(120→90) → 非コア機能 → 広告再評価
- **削らない項目**: プライバシーポリシー、利用規約、Data Safety、targetSdk/16KB対応、物理端末テスト、クローズドテスト、問題検証ツール

## 6. 参考: 問題JSONスキーマ（へぇの種）

```json
{
  "id": "science_0001",
  "question": "問題文",
  "options": ["選択肢A", "選択肢B", "選択肢C", "選択肢D"],
  "correctIndex": 0,
  "explanation": "解説",
  "category": "science",
  "difficulty": 1,
  "tags": ["宇宙"],
  "sourceTitle": "出典名",
  "sourceUrl": "https://example.com/",
  "checkedAt": "2026-07-18",
  "enabled": true
}
```

### 問題検証ツールの必須チェック
- ID空/重複チェック
- 選択肢4件、重複なし、correctIndex 0〜3
- 問題文・解説・出典が空でない
- カテゴリ許可一覧、難易度範囲
- 確認日形式
- 無効化問題除外
- 正解だけ不自然に長くないか
- 類似問題検出
- JSON全体のパース確認

**制作単位**: 10問作成 → 検証CLI → 誤字・出典確認 → 48h空けて再確認 → 10問単位コミット → 30問ごとにカテゴリ偏り確認
