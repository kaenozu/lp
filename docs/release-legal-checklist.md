## 3. 利用者データ

- [ ] 収集または処理するデータ種別を列挙した
- [ ] データごとの利用目的を記載した
- [ ] 端末外へ送信するデータと送信先を記載した
- [ ] 保存期間または削除条件を記載した
- [ ] 利用者が削除できる方法を確認した
- [ ] アカウント削除が必要な場合、その手順と受付方法を確認した
- [ ] バックアップ、ログ、キャッシュに残る範囲を確認した
- [ ] 子どもに関係する情報を扱う場合、対象年齢と保護者向け説明を確認した

## 4. LP法務ページ

- [ ] プライバシーポリシーが提出ビルドのSDK・通信・保存方式と一致する
- [ ] 利用規約の機能、料金、広告、課金に関する説明が実装と一致する
- [ ] 免責表現が一律の責任否定になっていない
- [ ] 問い合わせ先が有効である
- [ ] 制定日・最終改定日が正しい
- [ ] TODO、仮値、未確定表現が残っていない
- [ ] ストアから参照する法務URLがProductionでHTTP成功する

## 5. ストア申告

### Google Play

- [ ] データセーフティ回答が実装とプライバシーポリシーに一致する
- [ ] 広告の有無が一致する
- [ ] 対象年齢・コンテンツレーティングが実装と説明に一致する
- [ ] アプリ内課金、定期購入、価格表示が一致する
- [ ] プライバシーポリシーURLと問い合わせ先が正しい

### App Storeを使用する場合

- [ ] App Privacy回答が実装とプライバシーポリシーに一致する
- [ ] 年齢区分、課金、広告の申告が一致する
- [ ] Privacy Policy URLとSupport URLが正しい

## 6. LP表示と公開状態

- [ ] `tool/site_manifest.json`の`release_status`が実際の公開状態と一致する
- [ ] `published`の場合、正式なストアURLをmanifestへ登録した
- [ ] アプリLPの主要CTAがmanifestのストアURLを使用する
- [ ] ポータルカードの状態表示がmanifestと一致する
- [ ] `python3 tool/validate_site.py _site`が成功する
- [ ] Productionデプロイと全Statusが成功する

## 7. 証跡

次をリリースIssueまたはPRへ添付・リンクします。

- 依存SDK一覧または依存関係ファイル
- ネットワーク送信先の確認結果
- LP法務ページのProduction URL
- ストア申告の確認日と担当者
- リリース候補commit
- CI／Production監査の成功run

## 完了条件

未確認欄がなく、アプリ実装、LP、ストア申告の間に不一致がないこと。不一致がある場合は、コード、LP、申告のいずれかを修正してから提出します。
